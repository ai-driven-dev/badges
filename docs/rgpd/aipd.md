# Justification de non-réalisation d'une AIPD

Une AIPD (analyse d'impact, art. 35) est requise si le traitement présente un
risque élevé. Le CEPD (ex-G29, WP248) donne 9 critères ; une AIPD s'impose en
général dès **2 critères** réunis. Évaluation de ce traitement :

| # | Critère | Ici |
|---|---|---|
| 1 | Évaluation / scoring | Non — pas de profilage. La décision de certifier est humaine, hors de ce système. |
| 2 | Décision automatisée à effet juridique | Non — l'émission suit un merge décidé par un mainteneur. |
| 3 | Surveillance systématique | Non — aucune observation continue. |
| 4 | Données sensibles / hautement personnelles | Non — nom, LinkedIn, photo de profil, tous **fournis pour être publics**. |
| 5 | Grande échelle | Non — quelques dizaines/centaines de membres. |
| 6 | Croisement de jeux de données | Non. |
| 7 | Personnes vulnérables | Non — membres adhérents volontaires. |
| 8 | Technologie innovante à risque | Non — signature cryptographique standard (Open Badges 3.0). |
| 9 | Empêche l'exercice d'un droit / accès à un service | Non — l'adhésion et le retrait restent libres. |

**Critères réunis : 0.** Aucune AIPD n'est requise.

Le traitement porte de plus sur des données **publiques par conception**, avec un
droit d'effacement effectif (retrait). Cette justification est **datée et archivée**
dans le dépôt ; à réévaluer si le traitement change (échelle, finalité, nature des données).
